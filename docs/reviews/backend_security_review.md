# SortFlow 后端安全与边界评审（P1）

## 范围
- 路径白名单校验与文件系统访问
- 预览/执行流程中的目标路径安全
- 潜在路径穿越与符号链接风险

## 现状观察
- `ValidatePath` 使用绝对路径 + 前缀匹配，已处理根目录边界（避免前缀误判）。
- 文件列表、缩略图、元数据等接口在 handler 中做了路径白名单校验。
- 预览与执行接口未对目标路径进行白名单校验，执行引擎直接在文件系统中创建/移动文件。

## 主要问题
1. **执行目标路径缺少校验**：`/preview` 与 `/organize/execute` 未验证 `TargetPath` 是否位于允许根目录，存在越权写入风险。
2. **符号链接逃逸风险**：当前校验基于 `filepath.Abs` 与 `Clean`，未解析符号链接，可能在允许根目录下通过符号链接跳出。
3. **服务层缺少安全兜底**：文件系统操作主要在 service 层执行，若 handler 忘记校验，服务层缺少二次校验。

## 建议
- 对 `Preview`/`Execute` 的 `TargetPath` 引入白名单校验，防止写入非授权目录。
- 对 `SourcePath` 与 `TargetPath` 在执行引擎中加入校验或引入统一的安全门面，避免 handler 漏检。
- 对白名单目录与输入路径使用 `filepath.EvalSymlinks`（或限制符号链接），降低逃逸风险。

## 结论
路径白名单在部分接口已有覆盖，但执行链路与符号链接风险尚未完全闭合，需补齐安全校验与兜底措施。
