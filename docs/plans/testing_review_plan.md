# SortFlow 单测与代码评审计划总览

## 目标

1. 建立覆盖后端与前端的单元测试体系，保障核心逻辑可回归、可扩展，并可在 CI 中自动执行。
2. 根据已有方案文档（backend_plan.md / frontend_plan.md）对实现进行系统性 Review，确保功能、结构与约定一致。
3. 形成可重复的 Review 模板与问题清单，便于后续迭代复用。
4. 使用任务清单（CSV）进行进度跟踪与验收。

## 计划拆分

为便于独立推进前后端工作，本计划拆分为两个子计划与对应的任务跟踪表：

- **后端计划**：`testing_review_backend_plan.md`
- **前端计划**：`testing_review_frontend_plan.md`

对应的任务进度表：

- **后端任务表**：`testing_review_backend_tasks.csv`
- **前端任务表**：`testing_review_frontend_tasks.csv`
